TypeCasting - implicit (done by system) and explicit (done by programmer)


Implicit: 
Any Non-Primitive Datatype, concat to Primitive datatype gives String
String + Any Datatype = String
Num + Num/Null/bool = Num
Num + undefined => NaN (Not a Number whose value is zero) = num
bool + bool/null = Num
bool + undefined => NaN(Not a Number whose value is zero) = Num


Explicit: 
use constructors to intentionally convert the datatype to achieve a certain expected/controlled result
ex: Number( ), String( ), Boolean( )
Number( ) = 0
Number("") = 0
Number("Hi") = NaN
Number("123") = 123
Number(false/true) = 0/1
Number(null) = 0
Number(undefined) = NaN
Number([123.67]) = NaN
Number({j:8}) = NaN
And since NaN has value 0, the end result is always a Number

Booeanl( ) = false
Boolean("") = false
Boolean("abc") = true
Boolean("123") = true
Boolean(0) = false
Boolean(any number except 0) = true
Boolean (undefined) = false
Boolean(null) = false
Boolean([123,67]) = true
Boolean({j:8}) = true



Flow control Statement
JS executes code line by line. Flow control allows us to control how we want the program to run based on our logic. 

1. Conditional - if, if-else, if-else-if, nested if, switch
2. Loops - for, while (when exact no. of iterations is not known), do-while (executes min. once, and Then checks condition)

for-of => array
for-in => object


c. function - reusable block of code. 
parameter - variable to be used inside the function block of code. 

1. named function - 
function namedFunc(param){}

1. anonymous function - 
let anonymousFunc = function(param){}

1. arrow function
let arrowFunc = (param) => {}

Scope: Availability of something...
var - global scope
let - block scope
const block scope
variables defined inside a block of code, have pre-defined scope. within the function, var can exist outside the block it was defined in, while let & var cannot be called outside the blocks they are defined in. 

Hoisting: can be called before it is defined. var and namedFunc can be called without throwing an error... 

IIFE - self-invoking function. does not need explicit invoking.
condition: should be called first inside the file. top priority. 
(function(param) {})(param)
ex: commonly used in eCommerce websites to show sales popups...

Higher order func & callback func - 
when func inside func, the inside func will be resolved first, and is called a callback func. the outside func is the higher order func. 

generator function - 
function* gen(){
yield "10% cashback"
yield "100% cashback"
yield "50% cashback"
yield "better luck next time"
}
let genVal = gen()
console.log(genVal)
genval, being primitive datatype, will have to store more than one value because of the function. How do we resolve this? 
console.log(genVal.next());
output: {value: "10% cashback", done: false}
here done tells us if there's another value next or not. 
final output: {value: undefined, done: true}

Data Structures: 
Spread operator: ... - to combine or join multiple values into a single DS. appears on right side of equation. 
Array Spread: [...arr] - for arrays
Object spread: {...obj} - for objects
rest operator: (...args) - for functions

Destructure: let [x,y] = arr - to separate multiple values into a single value. appears on left side of equation. 

Array Advance Concepts: 
Homogenous - same type of values can be stored
Heterogenous - different types of values can be stored
flexible - add & del values using methods
a. Array Manipulation

1. push( ), add value to end
2. pop( ), remove value from end
3. shift( ), remove value from start
4. unshift( ), add value from start
5. splice( ), add & remove same time
arr.splice( startIndex, deleteCount, addValueOptional)

b. Array Merge

1. concat( ), join arrays
2. slice( ), select part of array - arr.slice(startIndex. endIndex+1)
3. fill( ), fills the array with a value, replace... same or diff values - arr.fill(value, startIndex, endIndex + 1)
4. includes( ), checks for value to search in array
5. indexOf( ), find first occurrence index no of given value from array
6. lastIndexOf( ), find last occurrence index no of given value from array
7. sort( ), arrange values in asc to desc order or vice versa with reverse( ). drawback - only works with single digit values. Because it works on ASCII table??
8. flat( ), destructure nested array into single array - flat(Infinity) - How many nested
